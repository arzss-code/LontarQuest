## Project

LontarQuest ("Dimensi Lontar") is a 2.5D top-down action-RPG roguelite built in **Unity 6 (6000.3.11f1)** with the **Universal Render Pipeline (2D)**. Single-player, PC/Windows. Player character is **Saka**. Most gameplay code, comments, and Inspector tooltips are written in **Indonesian** — match that language when editing existing scripts or writing user-facing/Inspector text.

## Building, Running, Testing

There is no CLI build/test pipeline — this is a standard Unity project driven from the Editor:

- **Open / run:** open the project in Unity 6000.3.11f1 and press Play on a scene under `Assets/Scenes/`. Main gameplay scenes: `Stage1.unity`, `Stage2.unity`, `TempleScene.unity`; flow/menu scenes under `Assets/Scenes/Tristan/` (`StartMenu`, `Prologue-1`, `Story-1`, `Cutscene_to_1-1`, `InsideTemple`).
- **Tests:** Unity Test Framework (`com.unity.test-framework`) is installed but no test assemblies/`.asmdef` exist yet. Run/author tests via **Window > General > Test Runner** in the Editor.
- **Solution files** (`*.sln`, `*.csproj`) are git-ignored and regenerated by Unity — never hand-edit or commit them.
- **Editor automation:** `LontarQuest` menu in the Editor menu bar (see `Assets/Scripts/Editor/LontarQuestSetupTool.cs`) scaffolds roguelite Data folders, sample Boon/Lore ScriptableObjects, and base prefabs.

## Architecture

### Roguelite Boon system (core loop)
Player power-ups ("Boons") are data-driven and stack additively:

- **`BoonData`** (ScriptableObject, `Assets/Scripts/ScriptableObjects/`) defines a buff: typed `Lontara` / `Batak` / `Kawi`, with float stat modifiers (attack/move speed, damage reduction, health regen, extra stamina, elemental flag). Authored as assets, not code.
- **`PlayerModifier`** (on the Saka prefab, requires `PlayerStats`) holds the list of active Boons and exposes `TotalX` aggregates via `RecalculateModifiers()`. `AddBoon()` appends + recalculates; `ResetAllBoons()` is the roguelite death reset.
- **`PlayerStats`** reads those aggregates — e.g. `MaxStamina` and `TakeDamage()` query `GetComponent<PlayerModifier>()` at call time for extra stamina / damage reduction. **This is the integration point:** new Boon-affected stats are wired by adding a field to `BoonData`, summing it in `PlayerModifier.RecalculateModifiers()`, then consuming the aggregate where the stat is used.
- **`BoonDoor` / `BoonUIManager` / `BoonUIElement`**: walking through a `BoonDoor` opens blockades to the next room and calls `BoonUIManager.ShowBoonSelection()` to present a 3-choice picker.

### Room / stage flow
**`RoomManager`** (`Assets/Scripts/Stage-1/`) drives the per-room arena loop via 2D trigger colliders: player enters → locks `entryDoor`, wakes `enemiesInRoom`, sets a quest objective → polls each `Update` until all enemies are destroyed (`enemiesInRoom.RemoveAll(null)`) → `RoomCleared()` opens `exitDoors`/`nextRoomBlockades` and spawns a `GuideParticle` pointing at the exit. Enemies start disabled (`SetActive(false)`) and are activated on room entry. `EndPortal` / `BossArenaController` handle stage exit and the Kepala Kala boss.

### Global singletons
Several managers use the `static Instance` singleton pattern; check for one before reimplementing global state:
- **`QuestManager.Instance`** — `SetObjective(string)` / `CompleteCurrentObjective()` drive the on-screen objective TMP text.
- **`JournalManager.Instance`** — `DontDestroyOnLoad`, persists unlocked lore across scenes, toggled with **Tab** (pauses via `Time.timeScale`). Lore is `LoreData` ScriptableObjects unlocked through `LoreInteractable`.
- **`GameManager.Instance`** — lightweight cross-scene flags (e.g. `HasKey`).
- **`DamagePopupManager.Create(pos, amount, isPlayer)`** — static floating damage numbers.

### Conventions & gotchas
- **Input is legacy Input Manager** (`Input.GetAxisRaw`, `Input.GetKeyDown`, `Input.mousePosition`) despite `com.unity.inputsystem` being installed and an `InputSystem_Actions.inputactions` asset existing. Follow the legacy style already in use unless migrating deliberately.
- **Mixed physics dimensions:** movement uses 3D `Rigidbody` (`PlayerMovement`) while interaction/room logic uses 2D triggers (`Collider2D`, `OnTriggerEnter2D`) — this is intentional for the 2.5D presentation. Tag the player `"Player"`; trigger code checks `CompareTag("Player")`.
- Death (`PlayerStats.Die`) reloads the active scene by build index after a fade — keep scenes registered in Build Settings.
- Scripts are organized by zone/scene under `Assets/Scripts/` (`Start Area`, `Prologue-1`, `Stage-1`, `TempleScene`, `StartMenu`, `Systems`, `ScriptableObjects`). Cross-cutting/reusable systems live in `Systems/`.
- Third-party asset folders (`Assets/JMO Assets`, `Assets/RafaelMatos`, `Assets/Travis Game Assets`, `Assets/Arts`) are vendor/art content — avoid refactoring them.
